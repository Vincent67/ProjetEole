Méthode getEncrypt (SQLServerDataSource)

Retourne une valeur Boolean qui indique si la propriété de chiffrement est activée.

Syntaxe

public boolean getEncypt()

Valeur de retour

true si le chiffrement est activé. Sinon, false.

Notes

Si la valeur true est attribuée à la propriété de chiffrement, le Pilote JDBC Microsoft pour SQL Server fait en sorte que le serveur SQL Server utilise le chiffrement SSL (Secure Sockets Layer) pour toutes les données envoyées entre le client et le serveur, si un certificat est installé sur le serveur.

Si la propriété de chiffrement n'est pas spécifiée ou ne possède pas la valeur false, le pilote ne force pas SQL Server à prendre en charge le chiffrement SSL. Si l'instance SQL Server n'est pas configurée pour forcer le chiffrement SSL, une connexion est établie sans aucun chiffrement. Si l'instance SQL Server est configurée pour forcer le chiffrement SSL, le Pilote JDBC Microsoft pour SQL Server active automatiquement le chiffrement SSL lors de l'exécution sur une machine virtuelle Java (JVM) correctement configurée, sinon la connexion est interrompue et le pilote génère une erreur. Si la propriété de chiffrement n'est pas définie, la méthode getEncrypt retourne la valeur par défaut false.
